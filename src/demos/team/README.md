# Team Demos

Collaborative experiments and team work-in-progress. This is the space for exploring ideas together!

## Purpose

Use this folder for:
- 🤝 Multi-person projects ("Let's figure out data tables together")
- 🎨 Design critiques ("Exploring 3 approaches to filtering")
- 🎯 Client demos ("Mockup for Product team review")
- 🔬 Pattern explorations ("Trying out different loading states")
- 🚀 Prototypes for new features or workflows

## Who Can Add Demos?

Anyone on the team! This is a collaborative space.

## Quality Bar

- ✅ Working (doesn't have to be perfect or polished)
- ✅ Documented intent ("This explores X approach vs Y approach")
- ✅ Open for feedback and iteration
- ❌ No broken/non-running code (test before committing!)

## Workflow

1. Create a demo file in `team/` folder
2. Add a comment at the top explaining the purpose
3. Commit and push
4. Share with team for feedback
5. Iterate together
6. If it becomes a best practice, promote to `official/`

## Examples of Good Team Demos

```typescript
/**
 * Employee Directory - Exploring different table layouts
 * 
 * Status: WIP - trying out 3 different approaches
 * Contributors: @paul, @sarah
 * 
 * Goal: Figure out best pattern for employee list with filtering
 * Questions:
 * - Should filters be in sidebar or above table?
 * - Inline editing or modal?
 */
```

## Tips

- Use descriptive filenames: `employee-directory-v2.tsx` not `demo1.tsx`
- Add your name in comments if you're the primary owner
- Feel free to ask questions in comments
- Update READMEs with learnings as you go
