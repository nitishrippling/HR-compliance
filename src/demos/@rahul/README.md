# Official Demos

These are the blessed starter templates and best-practice examples for Pebble Playground.

## Quality Standards

- ✅ Documented with clear comments
- ✅ Follows Pebble best practices
- ✅ Team approved
- ✅ Safe to copy for new projects
- ✅ Showcases proper component usage and design patterns

## Current Official Demos

- **app-shell-demo.tsx** - Complete app layout template with navigation, sidebar, and content areas. This is the blessed starter template for creating new full-page demos.

## To Add a Demo Here

1. Create your demo in `src/demos/private/` or `src/demos/@yourname/`
2. Get feedback and iterate
3. Open a PR to move it to `official/`
4. Tag `@design-systems-team` for review
5. Team approval required before merging

## Guidelines

- Demos should be well-commented and explain *why* not just *what*
- Use theme tokens (never hardcode colors, spacing, etc.)
- Follow accessibility best practices
- Include common use cases and edge cases
- Keep demos focused on a single concept or pattern
